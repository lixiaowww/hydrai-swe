## 🎯 需要获取的真实数据

### **优先级1: 洪水预测必需数据**

#### 1.1 历史洪水事件数据
**用途**: 洪水预测模型训练和验证
**格式**: CSV/JSON
**必需字段**:
```
- flood_date: 洪水发生日期
- flood_level: 洪水水位(m)
- affected_area: 受影响面积(km²)
- damage_estimate: 损失估算($)
- trigger_factors: 触发因素(降水、SWE融化等)
- station_id: 监测站ID
- coordinates: 经纬度
```

**数据源建议**:
- Manitoba Hydro历史记录
- 加拿大环境部洪水数据库
- 地方政府应急管理记录
- 保险理赔数据

#### 1.2 扩展水文数据
**用途**: 提高径流预测准确性
**时间范围**: 2010-2024年 (与SWE数据对齐)
**必需字段**:
```
- date: 日期
- station_id: 水文站ID
- discharge: 流量(m³/s)
- water_level: 水位(m)
- precipitation: 降水(mm)
- temperature: 温度(°C)
```

**数据源建议**:
- HYDAT数据库扩展
- Manitoba Hydro实时数据
- 加拿大水资源调查

### **优先级2: 水质监测数据**

#### 2.1 水质监测站数据
**用途**: 水质变化分析
**格式**: CSV
**必需字段**:
```
- sample_date: 采样日期
- station_id: 监测站ID
- turbidity: 浊度(NTU)
- total_phosphorus: 总磷(mg/L)
- total_nitrogen: 总氮(mg/L)
- ph: pH值
- dissolved_oxygen: 溶解氧(mg/L)
- bacteria: 细菌数量(CFU/100mL)
- temperature: 水温(°C)
- coordinates: 经纬度
```

**数据源建议**:
- 加拿大环境部水质监测网络
- Manitoba水质监测项目
- 大学研究项目数据
- 地方政府监测数据

#### 2.2 雪水融化水质数据
**用途**: 雪水融化对水质的影响分析
**特殊要求**: 
- 春季融化期密集采样数据
- 融雪前后水质对比
- 不同SWE水平下的水质变化

### **优先级3: 扩展气象数据**

#### 3.1 历史气象数据扩展
**用途**: 与SWE数据时间范围对齐
**时间范围**: 2010-2023年
**必需字段**:
```
- date: 日期
- temperature_max: 最高温度(°C)
- temperature_min: 最低温度(°C)
- precipitation: 降水(mm)
- humidity: 湿度(%)
- wind_speed: 风速(m/s)
- pressure: 气压(hPa)
- coordinates: 经纬度
```

**数据源建议**:
- 加拿大环境部气象站数据
- NOAA历史数据
- 大学气象观测网络

#### 3.2 极端天气事件数据
**用途**: 极端天气场景分析
**必需字段**:
```
- event_date: 事件日期
- event_type: 事件类型(暴雨、高温、寒潮等)
- intensity: 强度等级
- duration: 持续时间(小时)
- affected_area: 影响区域
- impact_assessment: 影响评估
```

### **优先级4: 管理和决策数据**

#### 4.1 水资源管理决策案例
**用途**: 智能决策支持系统
**格式**: JSON/CSV
**必需字段**:
```
- decision_date: 决策日期
- scenario_type: 场景类型(洪水、干旱等)
- decision_actions: 采取的行动
- cost: 成本($)
- effectiveness: 效果评估
- lessons_learned: 经验教训
```

**数据源建议**:
- Manitoba Hydro运营记录
- 地方政府应急响应记录
- 学术研究案例
- 国际最佳实践案例

#### 4.2 基础设施数据
**用途**: 风险评估和决策支持
**必需字段**:
```
- infrastructure_id: 设施ID
- type: 设施类型(大坝、堤防、泵站等)
- capacity: 容量/能力
- condition: 状况评级
- coordinates: 位置坐标
- maintenance_history: 维护历史
```

## 🔍 数据质量要求

### **质量标准**
- **完整性**: 缺失数据 < 10%
- **准确性**: 数据验证通过率 > 95%
- **时效性**: 最新数据延迟 < 24小时
- **一致性**: 数据格式标准化
- **可追溯性**: 完整的数据来源记录

### **验证机制**
- 交叉验证不同数据源
- 时间序列一致性检查
- 空间分布合理性验证
- 专家审核确认
